# تقرير التبعيات

## نظرة عامة

يقدم هذا التقرير قائمة بالتبعيات الرئيسية لمشروع بصيرة، مع التركيز على حزم Python المستخدمة في تطوير الوحدات من 1 إلى 5.

## تبعيات Python

يعتمد مشروع بصيرة على مجموعة من حزم Python لإنجاز وظائفه المختلفة. فيما يلي قائمة بالتبعيات الرئيسية:

| التبعية        | الغرض                                        | الإصدار (تقريبي) |
| :------------- | :------------------------------------------- | :--------------- |
| `pytest`       | إطار عمل للاختبار                               | 7.x              |
| `pytest-asyncio` | دعم الاختبارات غير المتزامنة                   | 0.21.x           |
| `pytest-mock`  | توفير أدوات للمحاكاة الوهمية (mocking)         | 3.x              |
| `pytest-benchmark` | قياس أداء الكود                                | 4.x              |
| `pylint`       | أداة تحليل ثابت للكود للتحقق من الجودة         | 2.x              |
| `autoflake`    | أداة لإزالة الاستيرادات غير المستخدمة          | 1.x              |
| `tiktoken`     | مكتبة لترميز التوكنات لنماذج اللغة الكبيرة     | 0.x              |
| `uuid`         | لتوليد معرفات فريدة عالميًا (UUIDs)            | مكتبة قياسية      |
| `datetime`     | للتعامل مع التواريخ والأوقات                   | مكتبة قياسية      |
| `logging`      | نظام تسجيل الأحداث                             | مكتبة قياسية      |
| `asyncio`      | لكتابة كود متزامن باستخدام `async/await`       | مكتبة قياسية      |

## إدارة التبعيات

يتم إدارة التبعيات باستخدام `pip`، ويوصى بتثبيت جميع التبعيات المحددة في ملف `requirements.txt` (إذا كان موجودًا) لضمان بيئة متسقة.

## التبعيات الداخلية

بالإضافة إلى تبعيات Python الخارجية، يعتمد المشروع أيضًا على مكوناته الداخلية (الوحدات) التي تتفاعل مع بعضها البعض من خلال واجهات برمجة التطبيقات المحددة (APIs) وقنوات المراسلة.
